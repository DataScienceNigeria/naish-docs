Key learnings & principles

Source fidelity is non-negotiable: Outputs must be grounded strictly in project file content. Ahmad has explicitly corrected instances where Claude extrapolated beyond source documents or introduced figures that contradicted memo findings. Invented or unanchored numbers are not acceptable, even as placeholders, without explicit labelling and Ahmad's sign-off.
Resolve structural decisions before drafting: Ahmad's established working pattern is to align on architectural, scoping, and judgment-call questions before producing outputs. Skipping this step has been corrected in prior sessions.
GBB constraint framing: The political/mandate dimension of GBB hosting is a real constraint but must be framed diplomatically as strategic alignment in all external-facing documents.
Hybrid architecture precision: The hybrid model is two independent parallel servers, not a distributed cluster. The HGX single-baseboard constraint is a hard technical boundary to flag consistently.
Use cases and capacity: The single H200 node can support all seven identified priority use cases with remaining headroom; three use cases consume ~57% of node capacity, leaving ~43% headroom. This must not be overstated or extrapolated.
GPU-hour convergence: Inference workloads run perpetually; GPU-hour totals are structurally static over the three-year horizon, explaining convergence across use cases.
